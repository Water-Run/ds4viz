【UI 设计风格约束（必须严格遵守，可与其它编码 Prompt 自由组合）】

目标气质：
- 现代、严谨、简练、冷静；更像 IDE/开发者工具而非营销页或移动 App。
- Swiss 排版秩序 + Japanese 克制 + Fluent/WinUI3 的“材质/层级（surface）”思路。
- 信息密度高但不拥挤：靠网格/对齐/分组/留白，而不是靠大色块与装饰。

页面类型：
- 数据结构可视化 + 代码编辑器（Fleet/VS Code 工作区范式）。
- 默认只做浅色主题，不实现深色模式与主题切换。

布局骨架（优先采用，保持简单）：
- 顶部 Title/Command Bar（高度约 52px）：品牌/页面标题 + 命令搜索（可选）+ 1–2 个主操作（Run/Step）。
- 主体四区（可响应式折叠，但不要做复杂抽屉系统）：
    1) Activity Rail（约 52px，竖向图标按钮）
    2) Sidebar/Explorer（约 260–320px，树/文件列表）
    3) Editor（主区：等宽字体、带行号的轻量代码视图/textarea）
    4) Visualizer（右侧面板：SVG/Canvas + Inspector + 2–3 个按钮）
- 响应式：<1200px 可隐藏最右侧 Visualizer；<900px 可隐藏 Sidebar；保持逻辑简单。

图标（强制）：
- 仅使用 Material Icons / Material Symbols（开源）。
- 图标只表达功能，不做装饰；禁用 emoji；禁用插画/卡通；禁用圆形头像。

色彩（Neutral-first + 单一 Accent，严禁 AI 常见审美）：
- 背景 Base：#f4f5f6（或相近的中性浅灰）
- “Mica-like” 材质层（用于顶栏/侧栏）：rgba(255,255,255,0.55) + blur（可用 backdrop-filter；但不要渐变背景）
- 内容岛 Islands（主要卡片/面板）：rgba(255,255,255,0.92) 或 #fff（优先白）
- 边框：rgba(0,0,0,0.08)；强调边框：rgba(0,0,0,0.14)
- 文本：主 #101010；正文 #3f3f3f；辅助 #737373
- Accent（仅一个，克制海军蓝）：#002060（hover 可略深 #0b2a78）
- 语义色（极少量、小面积标签/细边线）：成功 #22c55e / 警示 #f59e0b / 错误 #ef4444
- 严格禁止：蓝紫渐变、霓虹色、高饱和装饰色、彩虹配色、发光描边、大面积彩色铺底。

字体与排版（强制）：
- UI：Inter（400/500/600）；数字与代码：JetBrains Mono（等宽，对齐）。
- 8px 网格系统；组件间距取 8 的倍数。
- 分区标题（Panel Head）：12px、0.06em 字间距、全大写（或等效的小型标题风格）。
- 正文 12–13px；代码 12–13px 行高 1.5–1.7。
- 通过对齐、分组与留白控制密度；不要把标题做成 32px 的“大海报风”。

Surface 与层级（Fluent 风格的关键，高级感来源）：
- 优先用“边框 + 轻阴影 + 轻背景差”塑造层级，而不是重阴影。
- 阴影极克制：
    - 静态：0 1px 2px rgba(0,0,0,0.04) 或无
    - hover：0 10px 26px rgba(0,0,0,0.10)（仅在卡片/按钮 hover 时）
- 圆角：主容器 14px；小控件 10–12px；pill 999px。

交互与动效（必须克制、工具化）：
- 只做 micro-interactions：hover/focus/pressed/selected。
- 动效时长：120–160ms（hover/press），最多 240ms（切换）；缓动使用 ease/标准曲线。
- hover 上浮 1px；press 回落；禁止弹跳/大缩放/长动画（>0.3s）。
- Focus ring 必须清晰但克制（2–3px，中性灰或 Accent 透明版本）。
- 禁止：发光效果、强烈 hover 阴影跳变。

编辑器与可视化（实现保持轻量）：
- Editor：轻量代码呈现（textarea 或 pre/code），行号列固定宽 40–48px；语法高亮可“极简/低彩度”（keyword 用 Accent、comment 用 muted）。
- Visualizer：SVG 优先；网格线极淡 rgba(0,0,0,0.04)；节点/边灰阶；“当前节点/当前步骤”用 Accent 透明铺底 + 细描边。
- Inspector：键值对为主；数值等宽字体右对齐；不做复杂图表库与大组件系统。

可访问性（默认必须）：
- 键盘可用；Tab 焦点可见；点击区域高度 ≥30px；按钮高度 34–38px。
- 文本对比度足够；避免过浅灰正文。

实现约束（保持简单）：
- 结构清晰、组件数量少、可维护；避免“为了炫技”引入复杂状态管理/庞大框架。
- 不要生成长篇无意义的 UI；只实现上述骨架与必要控件即可。

输出要求：
- 生成的代码必须完全遵循以上风格约束；如与其它功能性需求冲突，优先保持视觉/交互约束一致性。
