【设计风格 Prompt：IDE/开发者工具（Fluent Surface × Material Shape/Motion），Light-only】

目标气质（必须）：
- 现代、严谨、简练、冷静；更像 IDE/开发者工具，而非营销页/移动 App。
- Swiss 排版秩序 + Japanese 克制；信息密度高但不拥挤，靠网格/对齐/分组/留白实现。
- 视觉融合：Fluent 的“surface/材质层级” + Material 的“shape/组件一致性 + motion 编排”，整体更现代但不花哨。

色彩（Neutral-first + 单一 Accent；必须更有层次但不花）：
- Base 背景：中性浅灰（建议 #f4f5f6 附近），允许微冷调（偏蓝灰）以提升现代感。
- Islands/内容容器：优先不透明白（#ffffff）或接近白的实色（例如 #fbfbfc）；避免大面积透明铺底。
- Mica-like 层（仅少量区域可用）：rgba(255,255,255,0.55) + blur；禁止渐变背景。
- 边框：rgba(0,0,0,0.08)；强调边框：rgba(0,0,0,0.14)。
- 文本：主 #101010；正文 #3f3f3f；辅助 #737373。
- Accent（仅一个，改为 Azure 蓝系，二选一，整站统一）：
  A) Azure Blue：#0078D4（hover #106EBE / pressed #005A9E）
  B) Fluent Blue：#0067B8（hover #005A9E / pressed #004578）
- Accent 的使用方式必须克制：主要用于交互态/选中态/关键指示；允许 Accent wash（例如 rgba(0,120,212,0.08)）用于选中背景，但不允许大面积纯色铺底。
- 语义色（极少量、小面积标签/细线/点）：成功 #22c55e / 警示 #f59e0b / 错误 #ef4444。
- 严禁：蓝紫渐变、霓虹、高饱和装饰色、彩虹配色、发光描边、大面积彩色铺底。

字体与排版（强制）：
- UI 字体：Inter（400/500/600）；数字与代码：JetBrains Mono。
- 8px 网格系统；间距只使用 8 的倍数。
- 标题体系：不做海报风大标题；以 12–14px 的小型分区标题 + 12–13px 正文为主。
- 分区标题（Panel Head）：12px、0.06em 字间距、全大写或等效小型标题风格。
- 代码：12–13px，行高 1.55–1.70；语法高亮低彩度（keyword 用 Accent，comment 用 muted）。

Shape / Surface（Material × Fluent 融合的关键）：
- 层级来自：轻背景差 + 边框 + 极克制阴影（不要厚重阴影）。
- 阴影极克制：
  - 静态：0 1px 2px rgba(0,0,0,0.04) 或无
  - hover：0 10px 26px rgba(0,0,0,0.10)（仅在卡片/按钮 hover）
- 圆角：主容器 14px；小控件 10–12px；pill 999px（Material 的一致 shape 但保持克制）。
- 可视化区域/图表/节点“底色必须不透明”：plot area、Canvas 背景、节点填充均用实色（#fff 或 #fbfbfc）。
  - 允许透明只用于“强调态 wash”（选中/当前步骤的淡洗色），但基础底不得透明。

图标（强制）：
- 仅 Material Icons / Material Symbols（开源）；图标只表达功能，不做装饰；禁用 emoji / 插画 / 卡通 / 头像。

组件视觉规则（通用，不绑定布局）：
- 控件高度：30px+；按钮建议 34–38px；点击区域明确。
- 选中/聚焦：focus ring 必须清晰（2–3px，使用 Accent 透明版本或中性灰）。
- 列表/树/表格：强调“行分隔 + 轻 hover + 明确选中态”，避免大色块。
- 标签/Chip：使用中性底 + 边框；仅在必要时用语义色的小点/细边。

数据可视化（Material-like 但低饱和）：
- 网格线极淡 rgba(0,0,0,0.04)；轴与文字使用中性灰。
- 系列配色优先：1 个 Accent + 少量语义色点缀（禁止多彩）。
- 当前节点/当前步骤：Accent wash（淡背景）+ Accent stroke（细描边）+ 轻微对比提升；不使用发光。
- 过渡时使用 opacity/transform/stroke 的短动画，避免闪烁和跳变。

动效与交互（更丰富，但仍工具化克制；必须）：
- 动效分层：micro-interactions + 状态切换编排 + 可视化步骤推进。
- 动效时长：
  - hover/press：120–160ms
  - 组件展开/收起/切换：180–260ms
  - 页面级状态切换（例如视图切换）：200–320ms（仅 opacity + 轻位移 4–8px）
  - 可视化 step 推进：160–240ms（节点/边的 stroke、fill、opacity 平滑过渡）
- 缓动：cubic-bezier(0.2, 0, 0, 1)（统一使用，禁止弹跳）。
- 允许更“现代”的动效编排（Material 思路）：
  - Fade-through：先淡出旧内容再淡入新内容（短、克制）
  - Shared-axis：同一语义层级的切换用轻位移（4–8px）+ 透明度
- 可选但克制的“点击反馈”（二选一）：
  - 轻微 ripple（不扩散过大、不发光、180ms 内结束），或
  - 轻微背景 wash + 1px 上浮/回落（仍然优先）
- 必须支持 prefers-reduced-motion：显著减少/禁用位移动画。

可访问性（默认必须）：
- 键盘可用；Tab 焦点可见；文本对比度足够；避免过浅灰正文。
- 表意优先：不要用颜色作为唯一信息通道（辅以边框/图标/文本）。

禁止项（保持“冷静工具感”）：
- 禁止渐变大背景、霓虹、发光描边、玻璃拟物过度、长动画（>0.35s）、夸张弹跳缩放。
- 禁止营销式 hero、大标题海报风、装饰插画与情绪化图形语言。

输出要求：
- 生成结果必须严格遵循上述色彩/shape/surface/motion 约束。
- 如功能需求与风格冲突：优先保持“中性克制 + 单一 Azure Accent + Fluent×Material 融合 + 不透明可视化底色 + 现代短动效”的一致性。
