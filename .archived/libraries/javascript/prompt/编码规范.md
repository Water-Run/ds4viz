# `ds4viz`: JavaScript编码规范

在编写JavaScript代码时，你需要遵循以下编码规范：

**语言级别:** `ECMAScript 2022+` (ES13+)，使用现代JavaScript特性。

**类型注解:** 使用JSDoc进行类型注解，提供完整的类型信息。虽然JavaScript是动态类型语言，但通过JSDoc可以获得IDE智能提示和基本的类型检查。在VSCode中启用`checkJs`获得类型检查。

**编码规范:** 遵循以下标准：
- 使用ESLint + Prettier进行代码检查和格式化
- 推荐使用Airbnb JavaScript Style Guide或Standard Style
- 在VSCode/WebStorm最严格模式下，不能有任何Warning或Error
- 采用现代ES6+写法，包括：
  - 优先使用`const`，需要重新赋值时使用`let`，禁用`var`
  - 使用箭头函数、解构赋值、展开运算符、可选链（`?.`）、空值合并（`??`）
  - 善用`map`、`filter`、`reduce`、`flatMap`、`forEach`等数组方法
  - 使用模板字符串而非字符串拼接
  - 使用`class`关键字定义类，支持私有字段（`#`）
  - 使用`async/await`处理异步操作
  - 使用对象字面量简写、计算属性名
  - 使用默认参数、剩余参数
  - 使用`Symbol`创建唯一标识符

**命名规范:**
- 文件名：使用kebab-case（如`data-processor.js`）
- 类名：使用PascalCase（如`DataProcessor`）
- 函数名、变量名、方法名：使用camelCase（如`processData`、`userName`）
- 常量：使用UPPER_SNAKE_CASE（如`MAX_RETRY_COUNT`）
- 私有成员：使用`#`前缀（如`#privateField`）

**文档字符串（JSDoc）:** 在文件开头、任何函数、类等都要有遵循以下规范的JSDoc文档注释。代码即文档的编码规范：代码需要尽可能自描述、易懂，减少非JSDoc之外的注释内容。

文件开头：

```javascript
/**
 * 文件功能描述，使用中文
 *
 * @file 文件名，从项目目录开始的相对路径
 * @author WaterRun
 * @date 时间，如2025-12-25。更新为最新时间。
 */
```

函数及方法：

```javascript
/**
 * 函数功能描述，使用中文
 *
 * @param {ParamType} paramName - 参数说明
 * @returns {ReturnType} 返回值说明
 * @throws {ErrorType} 异常说明
 *
 * @example
 * const result = functionName(arg1, arg2);
 */
```

类：

```javascript
/**
 * 类描述，中文
 *
 * @template T - 泛型参数说明（用于文档目的）
 */
```

类型定义（使用JSDoc typedef）：

```javascript
/**
 * 类型描述，中文
 *
 * @typedef {Object} TypeName
 * @property {string} prop1 - 属性1说明
 * @property {number} prop2 - 属性2说明
 */
```

枚举模拟（使用Object.freeze）：

```javascript
/**
 * 枚举描述，中文
 *
 * @enum {string}
 * @readonly
 */
```

属性：

```javascript
/**
 * 属性描述，中文
 *
 * @type {PropertyType}
 */
```

回调函数类型：

```javascript
/**
 * 回调函数描述，中文
 *
 * @callback CallbackName
 * @param {ParamType} param - 参数说明
 * @returns {ReturnType} 返回值说明
 */
```

**模块化:**
- 使用ES6模块系统（`import`/`export`）
- 优先使用命名导出，避免默认导出
- 一个文件一个主要类/功能，相关的辅助函数可以在同一文件
- 使用绝对路径导入（配置构建工具的alias）

**错误处理:**
- 定义自定义错误类继承自`Error`
- 使用`try-catch`进行错误处理，特别是异步操作
- 对于Promise，使用`.catch()`或`try-catch`配合`async/await`
- 验证函数参数，及时抛出有意义的错误

**代码示例:**

```javascript
/**
 * 数据处理工具类示例
 *
 * @file src/utils/data-processor.js
 * @author WaterRun
 * @date 2025-12-25
 */

/**
 * 数据项类型定义
 *
 * @typedef {Object} DataItem
 * @property {string} id - 数据项唯一标识
 * @property {string} name - 数据项名称
 * @property {number} value - 数据项数值
 */

/**
 * 处理结果类型定义
 *
 * @typedef {Object} ProcessResult
 * @property {boolean} success - 是否成功
 * @property {DataItem[]} [data] - 处理后的数据
 * @property {string} [error] - 错误信息
 */

/**
 * 过滤条件回调函数
 *
 * @callback FilterPredicate
 * @param {DataItem} item - 数据项
 * @returns {boolean} 是否保留该项
 */

/**
 * 数据处理状态枚举
 *
 * @enum {string}
 * @readonly
 */
const ProcessStatus = Object.freeze({
  PENDING: 'pending',
  PROCESSING: 'processing',
  COMPLETED: 'completed',
  FAILED: 'failed',
});

/**
 * 数据处理器类
 */
class DataProcessor {
  /** @type {DataItem[]} */
  #data;

  /** @type {ProcessStatus} */
  #status;

  /**
   * 构造函数
   *
   * @param {DataItem[]} data - 初始数据
   */
  constructor(data) {
    this.#data = data;
    this.#status = ProcessStatus.PENDING;
  }

  /**
   * 获取当前状态
   *
   * @returns {ProcessStatus} 当前处理状态
   */
  get status() {
    return this.#status;
  }

  /**
   * 过滤数据
   *
   * @param {FilterPredicate} predicate - 过滤条件函数
   * @returns {DataItem[]} 过滤后的数据
   *
   * @example
   * const filtered = processor.filter(item => item.value > 10);
   */
  filter(predicate) {
    return this.#data.filter(predicate);
  }

  /**
   * 映射转换数据
   *
   * @template T
   * @param {(item: DataItem) => T} mapper - 映射函数
   * @returns {T[]} 转换后的数据
   */
  map(mapper) {
    return this.#data.map(mapper);
  }

  /**
   * 异步处理数据
   *
   * @param {(item: DataItem) => Promise<DataItem>} asyncProcessor - 异步处理函数
   * @returns {Promise<ProcessResult>} 处理结果
   * @throws {Error} 处理过程中的错误
   */
  async processAsync(asyncProcessor) {
    try {
      this.#status = ProcessStatus.PROCESSING;
      const processedData = await Promise.all(
        this.#data.map(item => asyncProcessor(item))
      );
      this.#status = ProcessStatus.COMPLETED;
      return {
        success: true,
        data: processedData,
      };
    } catch (error) {
      this.#status = ProcessStatus.FAILED;
      return {
        success: false,
        error: error.message,
      };
    }
  }

  /**
   * 静态工厂方法
   *
   * @param {DataItem[]} data - 原始数据
   * @returns {DataProcessor} 处理器实例
   */
  static create(data) {
    return new DataProcessor(data);
  }

  /**
   * 静态辅助方法：验证数据项
   *
   * @param {unknown} item - 待验证的数据
   * @returns {item is DataItem} 是否为有效数据项
   */
  static isValidDataItem(item) {
    return (
      typeof item === 'object' &&
      item !== null &&
      'id' in item &&
      'name' in item &&
      'value' in item &&
      typeof item.id === 'string' &&
      typeof item.name === 'string' &&
      typeof item.value === 'number'
    );
  }
}

/**
 * 自定义错误类
 */
class DataProcessError extends Error {
  /**
   * 构造函数
   *
   * @param {string} message - 错误信息
   * @param {string} [code] - 错误代码
   */
  constructor(message, code) {
    super(message);
    this.name = 'DataProcessError';
    this.code = code;
  }
}

export { DataProcessor, ProcessStatus, DataProcessError };
```

**额外最佳实践:**

1. **不可变性**: 尽可能使用不可变数据结构，避免直接修改对象和数组
2. **纯函数**: 优先编写无副作用的纯函数
3. **参数验证**: 在函数开始处验证参数的有效性
4. **避免全局变量**: 使用模块作用域或闭包封装状态
5. **使用严格模式**: 在文件顶部添加`'use strict';`（ES6模块自动严格模式）
6. **链式调用**: 合理使用方法链提高代码可读性
7. **解构赋值**: 充分利用解构简化代码
