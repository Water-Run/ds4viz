# `ds4viz`: Lua编码规范

在编写Lua代码时，你需要遵循以下编码规范：

**语言级别:** `Lua 5.4+` 或 `LuaJIT 2.1+`  
**类型注解:** 虽然Lua是动态类型语言，但需要在文档注释中明确标注类型。如果使用LuaLS/EmmyLua，建议使用`---@type`等注解。编码时需要有类型意识，避免类型混乱  
**编码规范:** 遵循Lua官方风格指南。代码需要在LuaLS/Luacheck最严格模式下无Warning。采用现代Lua写法，包括：善用`local function`、闭包、元表、迭代器、多返回值、尾调用优化等Lua特性；使用`goto`进行错误处理（仅当必要时）；善用`...`可变参数、`select`、`table`库等；利用元表实现面向对象、运算符重载等高级特性  
**文档注释:** 在文件开头、任何的函数、类（元表）、模块上都要有遵循以下规范的文档注释。所有注释必须使用`--@`标签格式，不允许无标签的普通注释。代码即文档的编码规范：代码需要尽可能的自描述、易懂，减少非文档注释之外的注释内容。

## 命名约定

- **小写+下划线**: 变量名、局部变量、表字段 (如 `user_name`, `total_count`)
- **大写+下划线**: 常量 (如 `MAX_SIZE`, `DEFAULT_TIMEOUT`)
- **小驼峰**: 函数名、方法名 (如 `getUserName`, `calculateTotal`)
...遵循标准的lua规范  

## 文档注释规范

### 文件开头

```lua
--[[
文件功能描述，使用中文
多行描述可以继续写在这里
详细说明文件的用途和功能

Author:
    WaterRun
File:
    ds4viz/utils/data_processor.lua
Date:
    2025-12-23
Updated:
    2025-12-23
]]
```

### 模块定义

```lua
--[[
模块描述，中文
详细说明模块的功能和用途

Author:
    WaterRun
Module:
    DataProcessor
]]
local M = {}
```

### 函数和方法

```lua
--@description: 函数功能描述，使用中文
--@param param_name: type - 参数说明
--@param another_param: type|nil - 可选参数说明
--@return: type - 返回值说明
--@return: type|nil - 第二个返回值说明（如果有多返回值）
--@raise: error_type - 抛出错误的条件说明
--@usage: local result = functionName(arg1, arg2)
local function functionName(param_name, another_param)
    -- 实现
end
```

### 类（元表实现）

```lua
--[[
类描述，中文
详细说明类的用途和特性

Class:
    ClassName
Fields:
    field_name: type - 字段说明
    another_field: type - 另一个字段说明
]]
local ClassName = {}
ClassName.__index = ClassName

--@description: 构造函数描述
--@param param1: type - 参数说明
--@return: ClassName - 返回类实例
--@usage: local obj = ClassName.new(param1)
function ClassName.new(param1)
    local self = setmetatable({}, ClassName)
    self.field_name = param1
    return self
end

--@description: 方法描述
--@param self: ClassName - 类实例
--@param param: type - 参数说明
--@return: type - 返回值说明
function ClassName:methodName(param)
    -- 实现
end
```

### 常量定义

```lua
--@description: 最大批处理大小常量
--@const: MAX_BATCH_SIZE
local MAX_BATCH_SIZE = 100

--@description: 默认超时时间（秒）
--@const: DEFAULT_TIMEOUT
local DEFAULT_TIMEOUT = 30
```

### 表配置/数据结构

--@description: 配置表说明

local config = {

## 最佳实践

1. **使用local**: 所有变量默认使用`local`，避免全局污染
2. **模块返回**: 模块最后返回表，包含公开的接口
3. **元表实现OOP**: 使用元表实现类和继承
4. **错误处理**: 使用`assert`进行参数验证，`pcall`捕获错误
5. **多返回值**: 充分利用Lua的多返回值特性
6. **尾调用优化**: 在递归函数中使用尾调用
7. **迭代器**: 使用`pairs`、`ipairs`或自定义迭代器
8. **字符串操作**: 使用`string.format`而不是拼接
9. **表操作**: 使用`table`库函数进行表操作
10. **强制使用标签**: 所有文档注释必须使用`--@`标签，不使用无标签的注释
