# `ds4viz` 需求文档  

`WaterRun` `2025-12-19`  

## 概述  

`ds4viz` -> `data structure for visualization` -> `数据结构可视化`.  

考虑前后端分离的架构搭建网站, 页面设计参考力扣:  

- 顶部菜单栏  
- 左侧可视化  
- 右侧输入代码

点击执行 => 生成可视化结果.  
可视化结果可交互. 参考:  

- [algorithm-visualizer](https://github.com/algorithm-visualizer/algorithm-visualizer)

对于编写的代码部分, 提供对应语言的库`ds4viz`, 包含各种基本的数据结构.  
通过实例化库并调用对应方法实现数据结构的变更, 之所以要实现为单独的库的原因是, 在执行过程中`ds4viz`库将生成一个统一的`.toml`, 作为步骤变化的描述(可以理解为中间语言).  
可视化渲染器通过解析该`toml`("中间语言")实现可视化, 即:  

```plaintext
        [编码]                        [执行]                         [渲染器]
 调用`ds4viz`库的各种对象  ->   `ds4viz`库执行并生成TOML    ->    渲染器解析TOML生成可视化
```

对于这个网页, 则大致流程如:  

```plaintext
   [前端网页]                [后端服务器]                     [前端网页]
 编写代码, 提交     ->  执行, 生成toml, 验证有效性   ->  解析返回的toml, 渲染, 可视化 
 > 提供高亮, 补全       > 带限制(超时), 验证(避免恶意代码把系统炸了)
```

## 技术栈

- 前端 - 页面和可视化解析器: Vue3+TS, SPA
- 后端 - 代码执行和返回: Python, FastAPI, PostgreSQL  
- 库 - 生成通用可视化过程描述文件: 各个语言具体实现  

计划为以下语言提供`ds4viz`库支持:  

- Demo
    - `Python`(发布到`PyPi`上不用审核, 名字不被人抢了就行, 有发布库经验)  
- 完整实现进一步包括
    - `C`
    - `Zig`
    - `Rust`
    - `Java`
    - `C#`
    - `TypeScript`
    - `lua`


由于以统一的`.toml`作为描述数据结构变迁的中间语言, 该架构可以很容易的扩展:  

- 扩展语言支持: 编写对应语言的`ds4viz`库  
- 扩展可视化渲染器: 编写对应的对`.toml`的处理程序即可. `Rust`写的CLI工具, `C#`写的`WinUI3`Windows桌面端应用, 或`Dart`+`Flutter`的Android应用, 实现对应前端即可.

## 示例: 代码, 库的调用和生成的中间`.toml`  

示例1: 栈 `Python`  

```python
import ds4viz # ds4viz库, pip install ds4viz

stack = ds4viz.Stack() # 获取一个提供的标准栈实例

# 对栈执行操作
stack.push(10)
stack.push(20)
stack.pop()
stack.push(30)

# 保存结果
stack.commit()
```

中间`.toml`:  

```toml
# ds4viz trace: Stack 示例
# 目标: 用一份稳定的 toml 描述 "数据结构的状态变化" 与 "触发这些变化的代码位置"
# 说明: 渲染器只需要解析 toml, 就能复现每一步的状态, 并高亮右侧代码

# 元数据
[meta]
created_at = "2025-12-19T00:00:00Z"
lang = "python"
lang_version = "3.14.0"

# 包信息
[package]
name = "ds4viz"
lang = "python"
version = "1.0.0"

# 备注
[remarks]
title = "Stack demo" # 标题
author = "WaterRun"  # 作者
comment = "Python栈"  # 备注

# states: 只存 "状态快照"
# 说明: 每一个 state 都是一个完整可渲染状态, 渲染器无需回放所有操作也能直接展示

[[states]]
id = 0
items = []   # 栈内元素(从底到顶)
top = -1     # 栈顶索引(空栈为 -1)

[[states]]
id = 1
items = [10]
top = 0

[[states]]
id = 2
items = [10, 20]
top = 1

[[states]]
id = 3
items = [10]
top = 0

[[states]]
id = 4
items = [10, 30]
top = 1

# steps: 只存 "一次操作" 与它对应的 before/after 状态引用
# 说明: before/after 指向 states.id, 这样渲染端可以轻松做前进/后退/跳转
[[steps]]
id = 0
op = "push"               # 操作名(用于渲染器显示: push/pop 等)
args = { value = 10 }     # 参数(可扩展)
before = 0                # 操作前状态 id
after = 1                 # 操作后状态 id
code = { line = 5 }       # 对应源码行号(从 1 开始, 空行也计入)

[[steps]]
id = 1
op = "push"
args = { value = 20 }
before = 1
after = 2
code = { line = 6 }

[[steps]]
id = 2
op = "pop"
args = {}
ret = 20                  # 返回值(例如 pop 返回被弹出的元素)
before = 2
after = 3
code = { line = 7 }

[[steps]]
id = 3
op = "push"
args = { value = 30 }
before = 3
after = 4
code = { line = 8 }

# 执行成功时才有结果
[result]
commit_op = "commit"      # 收尾操作名
commit_line = 10          # commit 对应源码行号
final_state = 4           # 最终状态引用
ok = true                 # 是否成功
```

示例2: 二叉树(`C#`)  

```csharp
using System;
using Ds4Viz; // 包Ds4Viz, 通过 NuGet 引入

public static class Program
{
    public static void Main()
    {
        // 创建一个标准二叉树实例
        var bt = new BinaryTree<int>();

        // 构建: 手动插入一些节点
        bt.InsertRoot(10);
        bt.InsertLeft(10, 5);
        bt.InsertRight(10, 15);

        // 这里故意制造一个错误:
        // 对不存在的父节点(999)插入子节点, 触发运行时异常
        bt.InsertLeft(999, 1);

        // 保存结果(不会执行到这里)
        bt.Commit();
    }
}
```

中间`toml`:  

```toml
# ds4viz trace: BinaryTree 示例
# 目标: 用一份稳定的 toml 描述 "数据结构的状态变化" 与 "触发这些变化的代码位置"
# 说明: 渲染器只需要解析 toml, 就能复现每一步的状态, 并高亮右侧代码

# 元数据
[meta]
created_at = "2025-12-19T00:00:00Z"
lang = "csharp"
lang_version = "12.0"

# 包信息
[package]
name = "ds4viz"
lang = "csharp"
version = "1.0.0"

# 备注
[remarks]
title = "BinaryTree demo" # 标题
author = "WaterRun"       # 作者
comment = "C#二叉树(故意触发错误)" # 备注

# states: 只存 "状态快照"
# 说明: 每一个 state 都是一个完整可渲染状态, 渲染器无需回放所有操作也能直接展示
# 约定: 二叉树使用 nodes 数组描述, 每个节点用 id 唯一标识, left/right 为子节点 id 或 -1
[[states]]
id = 0
nodes = []                 # 空树
root = -1                  # 空树 root 为 -1

[[states]]
id = 1
nodes = [
  { id = 0, value = 10, left = -1, right = -1 }
]
root = 0

[[states]]
id = 2
nodes = [
  { id = 0, value = 10, left = 1, right = -1 },
  { id = 1, value = 5,  left = -1, right = -1 }
]
root = 0

[[states]]
id = 3
nodes = [
  { id = 0, value = 10, left = 1, right = 2 },
  { id = 1, value = 5,  left = -1, right = -1 },
  { id = 2, value = 15, left = -1, right = -1 }
]
root = 0

# steps: 只存 "一次操作" 与它对应的 before/after 状态引用
# 说明: before/after 指向 states.id, 这样渲染端可以轻松做前进/后退/跳转
# 注意: 出错的 step 只有 before, 没有 after
[[steps]]
id = 0
op = "insert_root"          # 插入根节点
args = { value = 10 }
before = 0
after = 1
code = { line = 13 }        # bt.InsertRoot(10);

[[steps]]
id = 1
op = "insert_left"          # 插入左子节点
args = { parent = 10, value = 5 }
before = 1
after = 2
code = { line = 14 }        # bt.InsertLeft(10, 5);

[[steps]]
id = 2
op = "insert_right"         # 插入右子节点
args = { parent = 10, value = 15 }
before = 2
after = 3
code = { line = 15 }        # bt.InsertRight(10, 15);

[[steps]]
id = 3
op = "insert_left"          # 故意触发错误: parent 不存在
args = { parent = 999, value = 1 }
before = 3
code = { line = 20 }        # bt.InsertLeft(999, 1);

# 执行失败时只有 error, 不应再出现 result
[error]
message = "Parent node not found: 999" # 错误信息
type = "runtime"                       # 错误类型(示例: timeout/runtime/validation/sandbox/unknown)
line = 20                              # 出错行号(对应触发错误的代码行)
```
