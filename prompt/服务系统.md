# 服务系统  

ds4viz提供在线Web服务. 在线Web服务使用Rust+Poem+PostgreSQL作为技术栈.  
后台管理系统在服务器的本地使用TUI, 采用lua+ltui.  
系统部署在Fedora 43.  
前端使用Vue SPA, 也由服务器返回.  

## 用户系统  

用户需要注册并登陆才能使用ds4viz.  
用户有两个级别: 默认的Standard级别和升级的Enhanced级别. 切换级别需要用户进行申请.  
用户可以填写头像和个人资料.  
用户可以被封禁等.  
在默认的Standard级别受约束:  

- 只能使用解释型语言  
- 每次刷新周期(有服务器配置定义)可用CPU时间(由服务器配置定义)限制, 默认为1小时/10s  
- 单次提交的代码最大不超过16KB  

Enhanced级别的用户没有约束.  

用户可以搜索另一个用户, 访问主页, 查看其最近执行的代码历史.  

## 代码运行  

使用systemd-run瞬态单元, 在临时沙箱中安全的运行代码.  
对于原有没有的.toml, 那新多的toml就是生成的toml--名称不一定.  
对于每次执行的历史结果, PostgreSQL 作为缓存系统. API需要返回是否缓存命中.  

代码有超时时间, 也可以在服务器中进行定义.  

语言暂时考虑支持以下, 但是要设计的高可扩展:  

- python(默认)  
- lua  
- rust  

## 模板系统  

系统提供一个模板系统, 包含指定示例.  
模板需要提供多个语言支持--包含系统支持的所有语言.  
每个模板包括示例代码和说明, 包含标题和正文, 使用markdown.  
模板可被用户评论, 点赞.  
用户可以收藏模板.
