## 0. 一句话目标

数据结构可视化教学平台。**用户用 Python / C 写一段调用 `ds4viz` 库的代码，库在执行时记录操作序列并生成 `.toml` 中间表示（IR），渲染器读取 `.toml` 交互式回放**。

核心架构：**代码 → `.toml` IR → 渲染**（跨语言、跨组件解耦）。

## 2. IR 契约（最高优先级）

**`.toml` 是跨语言/跨组件的硬契约**。改 IR 时必须同步所有语言库 + 渲染器 + 后端验证。

**修改前必读**：`document/IR定义.md`（约 950 行）。

### 2.1 顶层结构（固定顺序）

```toml
[meta]                # 必需: created_at, lang, lang_version
[package]             # 必需: name, lang, version
[remarks]             # 可选: title/author/comment

[object]              # 必需且唯一: kind, label?
                       # kind 必须是下列之一:
                       #   stack, queue, slist, dlist,
                       #   binary_tree, bst,
                       #   graph_undirected, graph_directed, graph_weighted

[[states]]            # 1..N, id 从 0 连续递增; data 字段由 object.kind 决定
[[steps]]             # 0..N, id 从 0 连续递增

[result]              # 成功 (final_state, commit?)
                       # 或
[error]               # 失败 (type, message, line?, step?, last_state?)
```

### 2.2 强约束速查

| 字段 | 约束 |
|------|------|
| `meta.lang` / `package.lang` | 必须一致；只允许 `python` / `c` |
| `meta.lang_version` | 仅 `0-9 a-z . + -` |
| `package.version` | 必须 `MAJOR.MINOR.PATCH` |
| `package.name` | 长度 1..64；仅 `0-9 a-z _ -` |
| `remarks.*` | 长度受限，不含换行 |
| `object.kind` | 9 个枚举值之一 |
| `states.id` / `steps.id` | 从 0 连续递增 |
| `value` 字段 | 仅 `int / float / str / bool` |
| 空指针 | 用 `-1` 表示 |
| `result` 与 `error` | 二选一，**禁止同时存在或都不存在** |
| 高亮 kind 适用 | stack/queue→`item`；含 nodes 结构→`node`；含 edges→`edge` |
| 无向图边 | 必须 `from < to`（库内部规范化） |

### 2.3 data 字段速查（按 object.kind）

| kind | data 字段 |
|------|----------|
| `stack` | `items` (scalar 数组), `top` (-1 / len-1) |
| `queue` | `items`, `front`, `rear` |
| `slist` | `head`, `nodes[]`（id, value, next, alias?） |
| `dlist` | `head`, `tail`, `nodes[]`（id, value, prev, next, alias?） |
| `binary_tree` / `bst` | `root`, `nodes[]`（id, value, left, right, alias?） |
| `graph_undirected` / `graph_directed` | `nodes[]`（id, label 1..32, alias?）, `edges[]`（from, to） |
| `graph_weighted` | 同上，edges 多一个 `weight` 字段 |

## 5. 公共概念与约定

### 5.1 错误类型（前后端两套，别混）

| 层 | 基类 | 位置 |
|----|------|------|
| Python 库 | `DS4VizError(RuntimeError)` / `StructureError` / `ValidationError`；枚举 `ErrorType = {runtime, timeout, validation, sandbox, unknown}` | `library/python/ds4viz/exception.py` |
| 后端 HTTP | `Ds4VizException` → 子类 `AuthenticationError/AuthorizationError/NotFoundError/ConflictError/ValidationError/ExecutionError/...` | `server/release/src/exceptions.py` |
| C 库 | `errmsg[512]` 字段 + `err_step/err_last_state/err_line`；error.type 固定为 `"runtime"` | `library/c/ds4viz.h` |

### 5.2 用户状态机

- `Active`：可登录、可执行代码、可收藏
- `Suspended`：可登录，**禁止**执行代码 / 收藏
- `Banned`：**禁止**登录

### 5.3 SupportedLanguage 单一事实源

`server/release/src/model/execution.py:25-32` 定义 `SupportedLanguage(StrEnum)`，值仅 `python` / `c`。  
**禁止** 在前端 `render/vue/src/types/api.ts` 或别处再写一份常量——后端模型是唯一来源，前端映射在 `api/` 层做 snake_case ↔ camelCase。

### 5.4 数据结构 API 命名约定

Python 和 C 库的 9 种结构提供等价能力。命名映射：

| 含义 | Python | C (短名 / 全名) |
|------|--------|------------------|
| 栈 | `dv.stack()` | `dvStack` / `ds4vizStack` |
| 队列 | `dv.queue()` | `dvQueue` / `ds4vizQueue` |
| 单链表 | `dv.single_linked_list()` | `dvSingleLinkedList` / `ds4vizSingleLinkedList` |
| 双向链表 | `dv.double_linked_list()` | `dvDoubleLinkedList` / `ds4vizDoubleLinkedList` |
| 二叉树 | `dv.binary_tree()` | `dvBinaryTree` / `ds4vizBinaryTree` |
| 二叉搜索树 | `dv.binary_search_tree()` | `dvBinarySearchTree` / `ds4vizBinarySearchTree` |
| 无向图 | `dv.graph_undirected()` | `dvGraphUndirected` / `ds4vizGraphUndirected` |
| 有向图 | `dv.graph_directed()` | `dvGraphDirected` / `ds4vizGraphDirected` |
| 带权图 | `dv.graph_weighted()` | `dvGraphWeighted` / `ds4vizGraphWeighted` |

高亮辅助：`dv.node()` / `dv.item()` / `dv.edge()` ↔ `dvNode` / `dvItem` / `dvEdge`。

### 5.5 上下文管理（保证 [result]/[error] 一定写出）

- Python：`with dv.stack() as s: ...` 走 `ContextManager.__exit__`（`session.py:413-446`）。
- C：`dvStack(s, "label") { ... }` 是 `for(...; !s.done; ...)` 宏，作用域结束自动调用 `ds4viz_p_stack_close_` → `ds4viz_p_flush_` 写 TOML。

## 6. 子项目详解（按改动频率排序）

### 6.1 `library/python`（高频改动区）

**核心文件**：
- `ds4viz/__init__.py` — 公共 API 入口（导出 `stack/queue/...` 工厂函数 + 高亮辅助）
- `ds4viz/session.py` — `Session` 与 `ContextManager`；每条操作记 step + state
- `ds4viz/writer.py` — 手工 TOML 序列化（**不**依赖第三方 TOML 库）
- `ds4viz/trace.py` — Trace / Step / State 等数据类
- `ds4viz/exception.py` — 错误类型
- `ds4viz/structures/_base.py` — 公共基类（`_build_data` 抽象方法）
- `ds4viz/structures/linear.py` — Stack / Queue / SList / DList
- `ds4viz/structures/tree.py` — BinaryTree / BST
- `ds4viz/structures/graph.py` — GraphUndirected/Directed/Weighted

**改 API 时的最小动作**：
1. 修改 `structures/...py` 的类方法
2. 同步 `__init__.py` 的工厂函数（如果新加方法）
3. 添加 `test/intergration_test.py` 中的端到端测试
4. 跑 `pytest -v test/intergration_test.py -k "新方法名"` 单点验证
5. 如果改了 `data` 字段，**必须**同步 IR 定义 + Vue 解析（见 §11.1）

**错误抛出约定**：
- 用户态错误（空栈 pop 等）→ `raise StructureError(message, line)`，`_base.py:146-156` 封装
- 库内部 bug → 抛普通 `RuntimeError`
- 长度/格式校验失败 → 抛普通异常（让 pytest 看到）

### 6.2 `library/c`（中频改动区）

**核心文件**：
- `ds4viz.h`（4 352 行）— 头文件包含**全部**实现（`#define DS4VIZ_IMPLEMENTATION` 开关）
  - 类型定义集中在前 600 行
  - 公共宏 + 高亮/步进宏 `DS4VIZ_P_CNTX_` + `ds4vizStep`/`ds4vizAmend`/`ds4vizPhase` 在 700-1000 行
  - 短名映射 `DS4VIZ_SHORT_NAMES` 在 1151-1231 行
  - 实现部分从 1242 行起：`ds4viz_p_flush_` (TOML 写出), 栈/队列/链表/树/图各自完整实现
  - 关键模式：`DS4VIZ_P_COMMON_FIELDS_` 宏保证所有结构体偏移量一致
- `test.c`（3 906 行）— 集成测试，每条结构一个 TestXxxIntegration 类

**改 C 库时的最小动作**：
1. 修改 `ds4viz.h` 对应 `_open_/_push_/_pop_/...` 函数
2. 同步 `DS4VIZ_P_COMMON_FIELDS_` 之外的字段（如果新加状态字段需保持偏移量）
3. 添加 `test.c` 中的测试
4. 跑 `make clean && make` 全量验证（C 编译对 `-Werror` 严格）
5. 如果改了 `ds4viz_p_*_ws_`（状态写入回调），**必须**同步 Python 端对应结构的 `_build_data`

**关键约束**：
- 所有结构体首字段必须是 `DS4VIZ_P_COMMON_FIELDS_`（用于 `offsetof` 跨结构访问）
- 错误路径使用 `DS4VIZ_P_ERR_(p, line, fmt, ...)` 宏，自动设置 `err=true` + `errmsg` + `err_step` + `err_last_state`
- `ds4viz_p_tval_` 序列化 `ds4viz_p_val_t` 为 TOML 值（int/float/string/bool）
- C 字符串值在出作用域前必须 `strdup` 到 `ret_pool`，close 时统一释放

- **首选**：`systemd-run --user --scope --property=MemoryMax=N M --property=CPUQuota=50% -- timeout --signal=KILL N <cmd>`（cgroup 硬隔离）
- **降级**：`timeout --signal=KILL N <cmd>`（Docker 环境检测到 `DBUS_SESSION_BUS_ADDRESS` / `XDG_RUNTIME_DIR` 缺失时自动切换）
- 检测逻辑：`_check_systemd_available()` (`sandbox_service.py:474-508`)

**沙箱注意**：
- 不会捕获孙子进程：`fork() + exec()` 可能逃逸 timeout（**生产必须**用 systemd-run + cgroup）
- C 编译命令：`gcc -std=c2x -Wall -Wextra -Wpedantic {filename} -o main -lm && ./main`
- Python：`python3 -u main.py`（`-u` 防止输出缓冲）
- 扫描工作区新增 `.toml` 文件作为输出（按字母序取首个）

### 6.5 `document/`（低频文档区）

- `IR定义.md` — IR 规范，**改 IR 前必读**
- `在线服务部署.md` — 部署命令

## 12. 关键设计决策（理解项目为何这样写）

### 12.1 三层解耦

代码 → `.toml` IR → 渲染。**任何一层不能直接依赖另一层的数据结构**。所有层只通过 `.toml` 文本通信。

### 12.2 上下文管理强制写盘

无论用户代码成功或抛异常，**必须**生成 `.toml`。Python 走 `ContextManager.__exit__`（`session.py:413-446`），C 走 `for` 宏自动调用 `close_`。这样渲染器永远有内容可显示。

### 12.3 states 是完整快照

每个 state 可独立渲染，不依赖其他 state。steps 是增量描述，渲染器需要通过 `before/after` 拼接。

### 12.4 高亮全量快照

每 step 的 `highlights` 是该步完整的高亮集合（不是增量）。这样渲染器无需回溯。

### 12.5 沙箱两层降级

systemd-run 优先（cgroup 硬隔离），无 systemd 时降级为 `timeout`（仅信号级超时）。**生产必须**配置 systemd。

### 12.6 单一事实源

- 语言枚举：`server/release/src/model/execution.py:SupportedLanguage`
- IR 允许的 kind：`document/IR定义.md` + 各解析器硬编码的 `ALLOWED_KINDS`
- IR 允许的 lang：`ALLOWED_LANGS = {python, c}`（跨多份同步）
- 别名格式：`/^[a-zA-Z0-9_-]+$/`（Python `_base.py:19`，C 宏 `DS4VIZ_P_COMMON_FIELDS_`，前端 `ir.ts:83`）

### 12.7 状态写入回调 (`*_ws_`)

C 库中每个结构体有一个 `ws_fn` 回调（`DS4VIZ_P_COMMON_FIELDS_` 中的 `ws_fn` 字段），在每次写 state 时被调用。**新增结构必须**实现这个回调，保证 `data` 字段序列化为合法 TOML。

### 12.8 错误时 error 块

C 库与 Python 库在异常路径上必须生成 `[error]` 块而不是空文件（`session.py:428-444`、`ds4viz.h:2278-2295`）。

def rotate(self, k: int) -> None:

    if k == 0:

### 16.3 Walkthrough C — 排查 "Vue 解析 IR 报 null"

**症状**: `Playground` 显示可视化空白, 控制台报 `IR 解析为 null`.

**排查序列**:

```bash
# 1) 直接打开生成的 .toml, 看是否合法
cat trace.toml

# 2) 跑 Python 库的解析
cd library/python
python -c "import tomllib; tomllib.loads(open('trace.toml').read())"
# 如果报错, 是 TOML 格式问题, 回到 writer.py

# 3) 检查 IR 校验函数
grep -n "ALLOWED_KINDS" render/vue/src/utils/ir.ts
# 确认 [object].kind 在 9 个枚举值之内
```

**常见根因**:
- `meta.lang` 写成 `"Python"` (大写) — 必须是 `"python"`
- `object.kind` 写成 `"binary tree"` (带空格) — 必须是 `"binary_tree"`
- 节点 `id` 复用了已删除节点的 id — IR 定义禁止
- `error.type` 写成 `"Runtime"` — 必须是 `"runtime"`
