# 在线服务部署  

## 一键启动 (Docker, 推荐)

开发/演示环境推荐使用 Docker Compose 启动整个栈:

```bash
cp docker/.env.example .env       # 按需修改端口/密码
./docker/manage.sh up             # 构建并启动 postgres + backend + frontend
./docker/manage.sh logs           # 跟踪日志
./docker/manage.sh down           # 停止
./docker/manage.sh clean          # 完全清理(含数据库)
```

启动后访问:

- 前端 SPA: `http://localhost:5173`
- 后端 API: `http://localhost:10000` (经 nginx `/api/` 反代)
- 后端 OpenAPI 文档: `http://localhost:10000/docs`

Compose 文件位于 [`docker-compose.yml`](../docker-compose.yml). 各服务细节见 [`docker/`](../docker/).

> Docker 模式下沙箱自动降级为 `timeout` 命令 (不依赖 `systemd-run`). 生产环境的 systemd 资源限制需要在宿主机上配置.

## 手动部署 (传统方式)

### 环境  

测试于以下环境:  

- `Fedora 43`  
- `i5-13620H`  

### 基础准备  

设备上应当具备与以下测试环境兼容的基础运行时:  

- `python`: 3.14.2  
- `PostgreSQL`: 18.1  
- `bun`: 1.3.9  
- `node`: v24.9.0

### 目录与模块  

- 后端服务：`./server/release`  
- 前端 SPA：`./render/vue`  
- 管理 REPL：`./server/release/admin-repl`

### 启动后端服务

参阅[后端服务文档](../server/release/README.md).  
启动前, 确保[数据库](../server/release/docs/数据库环境准备.md)已经正确部署且就绪.  

### 启动前端服务

参阅[Vue SPA前端文档](../render/vue/README.md).  

### 管理 REPL  

独立命令行工具, 直接写数据库, 简单的后台管理.  
参阅[Admin Repl文档](../server/release/admin-repl/README.md).
