# `ds4viz`: C编码规范

在编写C代码时，你需要遵循以下编码规范：

**语言级别:** `C23`  
**类型注解:** C是静态强类型语言，所有变量必须明确声明类型。使用`const`、`restrict`、`volatile`等类型限定符。善用`stdint.h`中的固定宽度整数类型(`int32_t`、`uint64_t`等)、`stdbool.h`中的`bool`类型  
**编码规范:** 遵循经典K&R和GNU风格的混合体。使用Allman风格(大括号独占一行)。代码需要在GCC最严格模式(`-Wall -Wextra -Wpedantic -Werror`)下无Warning。采用传统C写法，包括：所有变量在块开头声明、使用`typedef`定义类型别名、使用函数指针、使用结构体和联合体、避免使用VLA(可变长度数组)、使用`goto`进行错误清理(仅限必要时)  
**内存管理:** 严格遵循"谁分配谁释放"原则，所有`malloc`/`calloc`必须有对应的`free`，避免内存泄漏和悬空指针  
**文档注释:** 在文件开头、所有函数、结构体、枚举、宏定义上都要有遵循Doxygen风格的文档注释。代码即文档的编码规范：代码需要尽可能的自描述、易懂，减少非文档注释之外的注释内容。

## 命名约定

- **小写+下划线**: 函数名、变量名 (如 `get_user_name`, `total_count`)
- **大写+下划线**: 宏定义、枚举常量、全局常量 (如 `MAX_SIZE`, `DEFAULT_TIMEOUT`)
- **小写+下划线+_t后缀**: typedef类型别名 (如 `data_processor_t`, `status_t`)
- **小写+下划线+_s后缀**: 结构体标签 (如 `struct data_processor_s`)
- **小写+下划线+_e后缀**: 枚举标签 (如 `enum status_e`)
- **小写+下划线+_u后缀**: 联合体标签 (如 `union value_u`)
- **匈牙利命名法**: 指针使用`p`前缀 (如 `p_data`)

## 文件结构

每个C源文件必须按以下顺序组织：

1. 文件头注释
2. 包含指令(`#include`)
3. 宏定义
4. 类型定义(`typedef`, `struct`, `enum`, `union`)
5. 静态全局变量声明
6. 静态函数声明
7. 外部函数实现
8. 静态函数实现

每个C头文件必须包含：

1. 文件头注释
2. 头文件保护符(`#ifndef`)
3. 包含指令
4. 宏定义
5. 类型定义
6. 函数声明
7. 头文件保护符结束(`#endif`)

## 文档注释规范

```c
/**
 * @file data_processor.c
 * @brief 数据处理器实现，提供各种数据转换和验证功能
 * 
 * 详细描述文件的功能和用途
 * 该文件实现了数据处理的核心功能
 * 
 * @author WaterRun
 * @date 2025-12-23
 * @version 1.0.0
 */
```

### 文件开头(头文件)

```c
/**
 * @file data_processor.h
 * @brief 数据处理器接口定义
 * 
 * 提供数据处理相关的类型定义和函数声明

 * @version 1.0.0

#ifndef DS4VIZ_DATA_PROCESSOR_H
#define DS4VIZ_DATA_PROCESSOR_H

/* 头文件内容 */

#endif /* DS4VIZ_DATA_PROCESSOR_H */
```

### 函数

```c
/**
 * @brief 函数功能简述，使用中文
 * 
 * 函数的详细描述，说明其功能、用途、注意事项等
 * 可以分多行描述
 * 
 * @param param_name 参数说明
 * @param another_param 另一个参数说明
 * @return 返回值说明，包括可能的返回值含义
 * @retval 0 成功
 * @retval -1 失败，参数无效
 * @retval -2 失败，内存分配失败
 * @note 特别注意事项
 * @warning 警告信息
 * @see related_function() 相关函数引用
 * @since 1.0.0
 */
int function_name(int param_name, const char *another_param);
```

### 结构体

```c
/**
 * @brief 结构体简述，中文
 * 
 * 结构体的详细说明，描述其用途和使用场景
 * 
 * @struct data_processor_s
 * @typedef data_processor_t
 */
typedef struct data_processor_s
{
    /** @brief 连接字符串 */
    char *connection_string;
    
    /** @brief 批处理大小 */
    size_t batch_size;
    
    /** @brief 当前状态 */
    status_t status;
    
    /** @brief 私有数据指针 */
    void *private_data;
} data_processor_t;
```

### 枚举

```c
/**
 * @brief 处理状态枚举
 * 
 * 定义数据处理过程中的各种状态
 * 
 * @enum status_e
 * @typedef status_t
 */
typedef enum status_e
{
    /** @brief 待处理状态 */
    STATUS_PENDING = 0,
    
    /** @brief 处理中状态 */
    STATUS_PROCESSING = 1,
    
    /** @brief 已完成状态 */
    STATUS_COMPLETED = 2,
    
    /** @brief 失败状态 */
    STATUS_FAILED = 3
} status_t;
```

### 联合体

```c
/**
 * @brief 通用值联合体
 * 
 * 可以存储不同类型的值
 * 
 * @union value_u
 * @typedef value_t
 */
typedef union value_u
{
    /** @brief 整数值 */
    int32_t int_val;
    
    /** @brief 浮点值 */
    double float_val;
    
    /** @brief 指针值 */
    void *ptr_val;
} value_t;
```

### 宏定义

```c
/**
 * @brief 默认批处理大小
 * 
 * @def DEFAULT_BATCH_SIZE
 */
#define DEFAULT_BATCH_SIZE 100

/**
 * @brief 最大缓冲区大小
 * 
 * @def MAX_BUFFER_SIZE
 */
#define MAX_BUFFER_SIZE 4096

/**
 * @brief 计算数组元素个数
 * 
 * @param arr 数组名
 * @return 数组元素个数
 * @warning 仅适用于静态数组，不适用于指针
 */
#define ARRAY_SIZE(arr) (sizeof(arr) / sizeof((arr)[0]))

/**
 * @brief 安全释放内存并置空指针
 * 
 * @param ptr 要释放的指针
 */
#define SAFE_FREE(ptr) \
    do \
    { \
        if ((ptr) != NULL) \
        { \
            free(ptr); \
            (ptr) = NULL; \
        } \
    } while (0)
```

### 全局变量

```c
/**
 * @brief 全局处理器实例
 *

 * 
 * @warning 非线程安全，多线程环境需要加锁

static data_processor_t *g_processor = NULL;
```

### 函数指针类型定义

```c

typedef void *(*transformer_fn_t)(const void *data, size_t size, void *context);
```

## 最佳实践

1. **Allman风格**: 大括号独占一行
2. **变量声明**: 所有变量在块开头声明
3. **内存管理**: 严格配对malloc/free，使用SAFE_FREE宏
4. **错误处理**: 使用返回值表示成功/失败，使用goto清理资源(必要时)
5. **const正确性**: 尽可能使用const修饰不变参数
6. **头文件保护**: 使用#ifndef/#define/#endif保护
7. **extern "C"**: 头文件支持C++编译器
8. **断言**: 使用assert()验证内部逻辑(仅用于调试)
9. **固定宽度类型**: 使用stdint.h中的类型
10. **文档完整性**: 所有公开接口都必须有完整的Doxygen注释
