# `ds4viz`: PHP编码规范

在编写PHP代码时，你需要遵循以下编码规范：

**语言级别:** `PHP 8.2+`  
**类型注解:** 使用严格类型声明(`declare(strict_types=1)`)，所有参数、返回值、属性都必须有类型声明。善用联合类型(`string|int`)、交集类型、nullable类型(`?string`)、mixed类型等PHP 8.0+特性  
**编码规范:** 严格遵循PSR-1、PSR-12和PSR-4标准。代码需要在PHP_CodeSniffer和PHPStan(level 9)最严格模式下无Warning。采用现代PHP写法，包括：使用constructor property promotion、named arguments、match表达式、null safe operator(`?->`)、nullsafe coalescing(`??`)、arrow functions、attributes、enum、readonly properties、fibers等PHP 8.0+新特性。善用命名空间、trait、接口、抽象类、匿名类等面向对象特性  
**PSR-4自动加载:** 命名空间必须遵循PSR-4规范，类名与文件名一致，目录结构与命名空间对应  
**文档注释:** 在文件开头、任何的方法、类、接口、trait、enum上都要有遵循PHPDoc标准的文档注释。代码即文档的编码规范：代码需要尽可能的自描述、易懂，减少非PHPDoc之外的注释内容。

## 命名约定

- **PascalCase**: 类名、接口名、trait名、enum名 (如 `DataProcessor`, `UserInterface`)
- **camelCase**: 方法名、函数名、属性名、变量名 (如 `getUserName`, `totalCount`)
- **UPPER_CASE**: 常量、enum case (如 `MAX_SIZE`, `DEFAULT_TIMEOUT`)
- **命名空间**: 遵循PSR-4，如 `Ds4viz\Utils\DataProcessor`

## 文件结构

每个PHP文件必须按以下顺序组织：

1. 开始标签 `<?php`
2. 严格类型声明 `declare(strict_types=1);`
3. 文件级PHPDoc注释
4. 命名空间声明
5. use语句（按字母顺序）
6. 类/接口/trait/enum定义

## 完整示例

```php
<?php

declare(strict_types=1);

/**
 * 数据处理器类，提供各种数据转换和验证功能
 * 该类封装了常用的数据处理操作，支持批处理、异步处理等功能
 *
 * @author WaterRun
 * @file src/Utils/DataProcessor.php
 * @package Ds4viz\Utils
 * @since 2025-12-23
 * @version 1.0.0
 */

namespace Ds4viz\Utils;

use Ds4viz\Contracts\ProcessorInterface;
use Ds4viz\Enums\ProcessingStatus;
use Ds4viz\Exceptions\ProcessingException;
use Psr\Log\LoggerInterface;

/**
 * 数据处理器
 * 提供数据转换、验证、批处理等核心功能
 *
 * @package Ds4viz\Utils
 * @author WaterRun
 * @since 2025-12-23
 *
 * @template TInput
 * @template TOutput
 */
final class DataProcessor implements ProcessorInterface
{
    /**
     * 默认批处理大小
     *
     * @var int
     */
    private const DEFAULT_BATCH_SIZE = 100;

    /**
     * 最大重试次数
     *
     * @var int
     */
    private const MAX_RETRIES = 3;

    /**
     * 当前处理状态
     *
     * @var ProcessingStatus
     */
    private ProcessingStatus $status;

    /**
     * 创建数据处理器实例
     *
     * @param string $connectionString 数据源连接字符串
     * @param int $batchSize 批处理大小
     * @param LoggerInterface|null $logger 日志记录器（可选）
     * @param array<string, mixed> $options 额外配置选项
     * @throws \InvalidArgumentException 当连接字符串为空时抛出
     */
    public function __construct(
        private readonly string $connectionString,
        private int $batchSize = self::DEFAULT_BATCH_SIZE,
        private readonly ?LoggerInterface $logger = null,
        private array $options = [],
    ) {
        if (empty($this->connectionString)) {
            throw new \InvalidArgumentException('连接字符串不能为空');
        }

        $this->status = ProcessingStatus::PENDING;
    }

    /**
     * 处理数据集合并返回处理结果
     *
     * @param array<int, TInput> $data 待处理的数据数组
     * @param callable(TInput): TOutput $transformer 数据转换函数
     * @return array{results: array<int, TOutput>, successCount: int} 包含处理结果和成功数量的关联数组
     * @throws \InvalidArgumentException 当data为空或transformer不可调用时抛出
     * @throws ProcessingException 当处理过程发生错误时抛出
     *
     * @example
     * ```php
     * $processor = new DataProcessor('mongodb://localhost:27017');
     * $result = $processor->processData(
     *     [1, 2, 3],
     *     fn(int $x): int => $x * 2
     * );
     * // $result = ['results' => [2, 4, 6], 'successCount' => 3]
     * ```
     */
    public function processData(array $data, callable $transformer): array
    {
        if (empty($data)) {
            throw new \InvalidArgumentException('data不能为空');
        }

        if (!is_callable($transformer)) {
            throw new \InvalidArgumentException('transformer必须是可调用的');
        }

        $this->status = ProcessingStatus::PROCESSING;
        $this->logger?->info('开始处理数据', ['count' => count($data)]);

        $results = [];
        $successCount = 0;

        foreach ($data as $index => $item) {
            try {
                $results[$index] = $transformer($item);
                $successCount++;
            } catch (\Throwable $e) {
                $this->logger?->error("处理第{$index}项失败", [
                    'error' => $e->getMessage(),
                    'item' => $item,
                ]);
                $results[$index] = null;
            }
        }

        $this->status = $successCount === count($data)
            ? ProcessingStatus::COMPLETED
            : ProcessingStatus::FAILED;

        return [
            'results' => $results,
            'successCount' => $successCount,
        ];
    }

    /**
     * 批量处理数据
     *
     * @param array<int, TInput> $data 待处理的数据数组
     * @param callable(TInput): TOutput $transformer 转换函数
     * @param int|null $batchSize 批大小，为null时使用实例默认值
     * @return array<int, TOutput> 处理结果数组
     * @throws ProcessingException 当批处理失败时抛出
     */
    public function batchProcess(
        array $data,
        callable $transformer,
        ?int $batchSize = null
    ): array {
        $batchSize ??= $this->batchSize;
        $total = count($data);
        $allResults = [];

        for ($i = 0; $i < $total; $i += $batchSize) {
            $batch = array_slice($data, $i, $batchSize, preserve_keys: true);
            $batchResult = $this->processData($batch, $transformer);
            $allResults = [...$allResults, ...$batchResult['results']];
        }

        return $allResults;
    }

    /**
     * 获取当前处理状态
     *
     * @return ProcessingStatus 当前状态枚举
     */
    public function getStatus(): ProcessingStatus
    {
        return $this->status;
    }

    /**
     * 设置批处理大小
     *
     * @param int $batchSize 新的批大小
     * @return self 返回当前实例以支持链式调用
     * @throws \InvalidArgumentException 当批大小小于1时抛出
     */
    public function setBatchSize(int $batchSize): self
    {
        if ($batchSize < 1) {
            throw new \InvalidArgumentException('批大小必须大于0');
        }

        $this->batchSize = $batchSize;
        return $this;
    }

    /**
     * 验证数据是否为空
     *
     * @param mixed $data 待验证的数据
     * @return bool 是否为空
     */
    private function isEmpty(mixed $data): bool
    {
        return match (true) {
            $data === null => true,
            is_array($data) => empty($data),
            is_string($data) => trim($data) === '',
            default => false,
        };
    }
}
```

### 枚举示例

```php
<?php

declare(strict_types=1);

/**
 * 处理状态枚举
 *
 * @author WaterRun
 * @file src/Enums/ProcessingStatus.php
 * @package Ds4viz\Enums
 * @since 2025-12-23
 */

namespace Ds4viz\Enums;

/**
 * 处理状态
 *
 * @package Ds4viz\Enums
 */
enum ProcessingStatus: string
{
    /**
     * 待处理状态
     */
    case PENDING = 'pending';

    /**
     * 处理中状态
     */
    case PROCESSING = 'processing';

    /**
     * 已完成状态
     */
    case COMPLETED = 'completed';

    /**
     * 失败状态
     */
    case FAILED = 'failed';

    /**
     * 获取状态的中文描述
     *
     * @return string 状态描述
     */
    public function label(): string
    {
        return match ($this) {
            self::PENDING => '待处理',
            self::PROCESSING => '处理中',
            self::COMPLETED => '已完成',
            self::FAILED => '失败',
        };
    }

    /**
     * 判断是否为最终状态
     *
     * @return bool 是否为最终状态
     */
    public function isFinal(): bool
    {
        return match ($this) {
            self::COMPLETED, self::FAILED => true,
            default => false,
        };
    }
}
```

## 最佳实践

1. **严格类型**: 始终使用 `declare(strict_types=1)`
2. **PSR-4自动加载**: 命名空间与目录结构严格对应
3. **不可变性**: 优先使用 `readonly` 属性和 `final` 类
4. **命名参数**: 在参数较多时使用命名参数提高可读性
5. **Match表达式**: 使用 `match` 替代 `switch`，更安全严格
6. **Null安全**: 使用 `?->` 和 `??` 处理可空值
7. **箭头函数**: 简单闭包使用 `fn()` 语法
8. **属性提升**: 构造函数参数直接声明为属性
9. **枚举优先**: 使用 `enum` 替代类常量组
10. **异常处理**: 使用类型化异常，遵循PSR异常接口
