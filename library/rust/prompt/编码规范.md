# `ds4viz`: Rust编码规范

在编写Rust代码时，你需要遵循以下编码规范：

**语言级别:** `Rust 2021 Edition` (Rust 1.70+)  
**类型注解:** Rust是静态强类型语言，编译器会进行类型推导，但在函数签名、结构体字段、公共API中必须明确标注类型。善用泛型、trait bounds、关联类型、生命周期参数等高级类型特性  
**编码规范:** 严格遵循Rust官方风格指南(rustfmt)和Clippy规范。代码需要在`cargo clippy -- -D warnings`最严格模式下无Warning。采用现代Rust习惯用法，包括：使用`?`运算符进行错误传播、使用`if let`/`while let`进行模式匹配、使用迭代器链式调用、使用`impl Trait`简化返回类型、使用`async/await`异步编程、使用宏(macro_rules!、过程宏)、使用`const fn`和`const generics`等。遵循所有权、借用、生命周期规则，避免不必要的克隆和分配  
**错误处理:** 使用`Result<T, E>`和`Option<T>`进行错误处理，避免panic(除非是不可恢复的逻辑错误)。使用`thiserror`或`anyhow`库优化错误处理  
**文档注释:** 在文件(模块)开头、所有公共(pub)函数、结构体、枚举、trait、宏上都要有遵循rustdoc标准的文档注释(三斜线`///`或模块级`//!`)。代码即文档的编码规范：代码需要尽可能的自描述、易懂，减少非文档注释之外的注释内容。

## 命名约定

- **snake_case**: 变量名、函数名、模块名、宏名、字段名 (如 `user_name`, `get_total`)
- **SCREAMING_SNAKE_CASE**: 常量、静态变量 (如 `MAX_SIZE`, `DEFAULT_TIMEOUT`)
- **PascalCase**: 类型名、trait名、枚举名、枚举变体 (如 `DataProcessor`, `ProcessingStatus::Pending`)
- **'a, 'b**: 生命周期参数 (如 `'static`, `'a`)
- **T, U, K, V**: 泛型类型参数 (如 `T`, `TResult`)

## 文件结构

每个Rust文件(模块)必须按以下顺序组织：

1. 模块级文档注释(`//!`)
2. 属性声明(`#![...]`)
3. use语句(按标准库、外部crate、当前crate分组，组内按字母顺序)
4. 模块声明(`mod`)
5. 常量定义
6. 类型定义(struct、enum、type alias、trait)
7. trait实现
8. 函数定义

/// 数据处理器结构体
///

///

### 枚举

```rust
/// 处理状态枚举
///
/// 定义数据处理过程中的各种状态
///
/// # 变体
///
/// * `Pending` - 待处理状态
/// * `Processing` - 处理中状态
/// * `Completed` - 已完成状态
/// * `Failed` - 失败状态，包含错误信息
///
/// # 示例
///
/// ```
/// use ds4viz::ProcessingStatus;
///
/// let status = ProcessingStatus::Pending;
/// assert!(status.is_pending());
/// ```
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum ProcessingStatus {
    /// 待处理状态
    Pending,
    
    /// 处理中状态
    Processing,
    
    /// 已完成状态
    Completed,
    
    /// 失败状态
    ///
    /// # 字段
    ///
    /// * `0` - 错误消息
    Failed(String),
}
```

### Trait

```rust
/// 数据处理器trait
///
/// 定义数据处理的核心接口
///
/// # 关联类型
///
/// * `Input` - 输入数据类型
/// * `Output` - 输出数据类型
/// * `Error` - 错误类型
///
/// # 必需方法
///
/// * `process` - 处理单个数据项
/// * `batch_process` - 批量处理数据
///
/// # 示例
///
/// ```
/// use ds4viz::Processor;
///
/// struct MyProcessor;
///
/// impl Processor for MyProcessor {
///     type Input = i32;
///     type Output = String;
///     type Error = std::io::Error;
///
///     fn process(&self, input: Self::Input) -> Result<Self::Output, Self::Error> {
///         Ok(input.to_string())
///     }
/// }
/// ```
pub trait Processor {
    /// 输入数据类型
    type Input;
    
    /// 输出数据类型
    type Output;
    
    /// 错误类型
    type Error;
    
    /// 处理单个数据项
    ///
    /// # 参数
    ///
    /// * `input` - 输入数据
    ///
    /// # 返回
    ///
    /// 成功返回处理后的数据，失败返回错误
    ///
    /// # 错误
    ///
    /// 当处理失败时返回`Self::Error`
    fn process(&self, input: Self::Input) -> Result<Self::Output, Self::Error>;
    
    /// 批量处理数据
    ///
    /// # 参数
    ///
    /// * `inputs` - 输入数据切片
    ///
    /// # 返回
    ///
    /// 成功返回处理结果向量，失败返回错误
    ///
    /// # 错误
    ///
    /// 当任一项处理失败时返回第一个错误
    fn batch_process(&self, inputs: &[Self::Input]) -> Result<Vec<Self::Output>, Self::Error>
    where
        Self::Input: Clone;
}
```

## 最佳实践

1. **所有权和借用**: 优先使用借用，避免不必要的克隆
2. **错误处理**: 使用`Result`和`?`运算符，避免panic
3. **类型安全**: 充分利用Rust的类型系统，使用`newtype`模式
4. **迭代器**: 使用迭代器链式调用，避免显式循环
5. **生命周期**: 明确标注生命周期参数，避免悬垂引用
6. **文档测试**: 在文档注释中包含可运行的示例代码
7. **must_use**: 对重要的返回值使用`#[must_use]`属性
8. **const fn**: 将可以在编译期求值的函数标记为`const`
9. **derive宏**: 充分利用`#[derive]`自动实现trait
10. **模块化**: 合理组织模块结构，使用`pub use`重导出
